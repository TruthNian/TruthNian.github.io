    ARP（Address Resolution Protocol，地址解析协议）是一种操作于网络层之下的协议，其作用是将IP地址转化为对应的MAC地址。在以太网中，数据帧通过MAC地址来标识目标设备，而IP地址则是用于跨网络进行通信。每个设备都有一个唯一的MAC地址，因此ARP协议可以解决IP地址到MAC地址的映射问题，从而确保数据包能够正确地发送到目标设备。

ARP协议工作原理如下：当一个主机需要发送数据到另一个主机时，它首先检查本地的ARP缓存，如果目标主机的IP地址已经在缓存中，则可以直接获取对应MAC地址。如果没有找到，则会在本地网络上广播请求包，询问具有目标IP地址的设备的MAC地址，该请求广播将被发送到所有设备，IP地址匹配的设备将会响应并回复自己的MAC地址。然后源主机就可以将MAC地址与IP地址映射起来，以后可以直接使用此缓存的记录进行通信。

  6. ×
    RIP（Routing Information Protocol）协议是一种较为简单的距离向量协议，其工作原理是通过跳数来计算最优路径，然后将路由信息传输给相邻的路由器。RIP协议最初的设计目的是用于中小型网络，而不是大型复杂网络。
    受限于协议本身的原理和限制，RIP协议在超过16台路由器组成的大规模网络中容易出现路由环路问题、路由信息更新不及时等问题，影响网络的稳定性和可靠性。

  1. 收发两端传输具体是1000KM，信号在媒体上的传播速率是2*10^8m/s，当数据长度为10^7bit，试计算数据发送速率为100kb/s时产生的发送时延和传播时延是多少？
    解：
    发送时延：T_s = L / R = 10^7 bit / 100000 bit/s = 100 s
    传播时延：T_p = D / v = 1000000 m / (2×10^8 m/s) = 5 ms

  3. 计算机ping外网IP地址，分析通信流程，产生报文类型及其作用。
    答：
    使用计算机第一次ping某外网ip地址时的通信流程：

    （2）查看ARP缓存表，有无网关IP的MAC地址，如果没有需要进行ARP请求获取网关MAC地址。

（2）ICMP询问报文：指定ICMP报文为回送请求与回送回答报文

  1. RIP协议中“关于坏消息传得慢”的理解
    在某些条件下，RIP协议对网络故障响应十分缓慢。比如：
    （1）若某个路由器相连的网络发生故障，该路由器中到达该网络的距离值被修改为16（不可达）
    （2）但此时有可能收到来自另一个路由器的路由信息，从而更新了该正确的距离值16为其他小于16的数值
    （3）路由器之间互相交换路由信息，直到该值重新变为16，才知道该网络已经出现了故障不可达，这个过程往往需要数分钟时间。

  2. TCP在建立连接时三次握手的过程以及为什么要采用三次握手。
    答：
    假定主机A和主机B通信。
    主机B运行服务器进程，被动打开并检测是否有客户进程的连接请求。
    （1）主机A向主机B发出连接请求；
    （2）主机B如同意则向主机A发回确认；
    （3）主机A收到主机B的确认后，要给主机B发出一个确认。
    采用三次握手的原因是防止已失效的连接请求报文又传送到主机B。

  3. 假定1公里长的CSMA/CD网络的数据率为2Gbps。设信号在网络上的传播速度为200000km/s。求能够使用此协议的最短帧长是多少字节。
    解：
    端到端时延：1km/200000km/s=5微秒
    来回的时延：2*5=10微秒
    最短帧长：2*5*10^-6*2*10^9/8=2500字节

    为了确定子网，我们需要知道子网掩码。/18表示子网掩码的前18位都是1，后14位都是0。所以，子网掩码是： 11111111 11111111 11000000 00000000 十进制表示为： 255.255.192.0 然后，我们可以用子网掩码和IP地址进行按位与运算，得到子网号。
    例如，对于10.10.100.10/18主机，我们可以将其IP地址和子网掩码分别转换为二进制表示： IP地址： 00001010 00001010 01100100 00001010 子网掩码： 11111111 11111111 11000000 00000000 按位与运算： 00001010 00001010 01000000 00000000 这就是该主机所属的子网号。转换为十进制表示为： 10.10.64.0
    我们可以看到，只有C选项的主机所属的子网号与之不同。

    DNS（Domain Name System）是一个应用层协议，用于将域名转换为 IP 地址。它依靠 UDP 协议来提供域名解析服务，使用UDP dport=53进行通信。DNS数据包通常比较小，且可以快速发送和接收，因此使用UDP可以提高性能。
    其它使用UDP协议的还有：网络时间协议（NTP）、远程过程调用（RPC）、网络文件系统（NFS）、简单网络管理协议（SNMP）、简单文件传输协议（TFTP）

  6. A
    电路交换（Circuit Switching）是指建立一条物理通路，通信双方在数据传输前，必须先建立一条完整的物理通路。在整个数据通信过程中，所分配的物理通路只供通信双方使用，即使在通信数据空闲期，所分配的通路也始终占用不释放。因此，电路交换的电路利用率相对较低，因为占用的通路不能被其他通信所利用，当某一通信时间很短时，接入通信线路了也很长时间无法被利用。

  7. B
    RARP的工作原理是，当一个只知道自己的物理地址而不知道自己的IP地址的站点初始化后，它会向本地网络上的所有主机发送一个RARP请求数据包，其中包含自己的物理地址。如果有一个RARP服务器收到了这个请求，它就会在自己的地址映射表中查找与该物理地址对应的IP地址，并将其放在一个RARP响应数据包中发送给请求者。这样，请求者就可以得到自己的IP地址了。

  8. D
    通信子网（互联网核心部分）是指负责数据传输的网络部分，它对应于 OSI模型中的低三层（物理层、数据链路层、网络层），而传输层是第四层，不属于通信子网的范畴 。
    传输层主要负责提供端到端的可靠数据传输服务，它在通信子网之上，为资源子网提供服务。

    假设一个B类地址为172.32.0.0-172.63.0.0，
    那么换成二进制地址为10101100:00100000:00000000:00000000-10101100:00111111:00000000:00000000
    可知，前11位相同，即172.32.0.0/11

  12. C
    直接进行 IP 封装的协议是 OSPF（Open Shortest Path First）。

OSPF 是一种基于链路状态算法的路由协议，它可以帮助构建一个 IP 网络环境下的路由信息传递体系。OSPF 协议中，路由器之间会通过信息交流来构建路由表，这些信息是封装在 IP 数据包中。因此，在 OSPF 协议中，信息可以直接通过 IP 封装来进行传输。

相比之下，RIP（Routing Information Protocol）和 BGP（Border Gateway Protocol）都是基于距离向量算法的路由协议。RIP 协议将路由信息封装在 UDP 数据包内进行传输，而 BGP 协议是将路由信息封装在 TCP 数据包内进行传输。

DNS（Domain Name System）是一种用于解析域名和 IP 地址之间对应关系的协议，它的数据封装在 UDP 或 TCP 数据包中传输，但它并不是一个路由协议，与题目所问的不符。

判断题

  1. √
    ISO 组织制定的 OSI 参考模型是一种网络分层架构，它将整个网络通信系统分成七个层次，每个层次执行特定的功能，并与其他层次相互通信。

  2. ×
    广域网（WAN）是指覆盖范围广、跨越城市乃至国家、大洲甚至全球的计算机网络。在广域网中，常用的拓扑结构包括星型拓扑、环型拓扑、网状拓扑等。
    总线型拓扑结构是一种使用同一媒体或电缆连接所有端用户的方式，这种结构在局域网中使用较多。

  3. ×
    双绞线确实是一种常见的传输介质，但并不是带宽最宽、抗干扰能力最强的类型。事实上，光纤等其他介质比双绞线有更高的带宽和更好的抗干扰能力。

  4. ×
    应用网关是一种 Web 流量负载均衡器，可以根据 HTTP 请求的属性（如 URI 路径或主机头）进行路由决策。应用网关是一种应用层（OSI 层 7）负载均衡器，而不是数据链路层（OSI 层 2）的设备。数据链路层实现网络互连的设备有交换机、网桥等。

  5. √
    密码学包括两个主要分支，一个是密码编码学，它研究的是如何设计和使用密码来加密和解密信息;另一个是密码分析学，它研究的是如何攻击和破解密码系统。

  6. √
    AES算法是一种公开的对称加密算法，在2001年被美国政府确定为美国政府使用的加密标准，其加密和解密过程都是公开的。但是，AES加密中使用的密钥是必须保密的，否则就会使加密过程变得毫无意义。密钥是加密解密的关键之一，只有找到正确的密钥，才能对加密的内容进行解密操作。因此，保护AES加密的密钥是非常重要的。

  7. ×
    一个C类地址由3个字节的网络地址和1个字节的主机地址组成，网络地址的最高位必须是“110”，范围从192.0.0.0到233.255.255.255。C类网络可达209万余个，每个网络能容纳254个主机。因为主机位不能全为0或全为1，否则表示网络地址或广播地址。所以，一个C类地址拥有8位主机位，可给254台主机分配地址，而不是256台。

  8. ×
    IGMP是网络层的协议，而不是传输层的协议。IGMP是Internet Group Management Protocol的简称，又被称为互联网组管理协议，是TCP/IP协议栈中负责IPv4 组播成员注册管理的协议。IGMP直接在IP层工作，每个IGMP报文都有一个IP头和一个IGMP头 。IGMP不使用传输层协议，如TCP或UDP。

  9. ×
    进程寻址使用的是端口号（Port Number）来实现。在网络通信过程中，每个应用程序通常都会绑定到一个端口号上，这个端口号是一个16位的数字，用于在网络连接中标识不同的进程。当两个进程需要进行通信时，需要通过端口号来确定彼此的身份，从而成功建立连接。因此，进程寻址是使用主机地址加上端口号来实现的，而不是使用端点（TSAP）标识。
    TSAP（Transport Service Access Point）是用于在OSI模型的表示层和会话层之间提供传输服务访问的标识符，它不直接用于进程寻址。

  10. √
    WWW (World Wide Web) 服务是一种分布式的、基于客户端/服务器 (Client/Server)方式工作的互联网服务。
    WWW客户端是一个通过发送HTTP请求到WWW 服务器上获取特定内容的程序，而WWW服务器负责响应这些请求并返回相关的内容。
    因此，WWW服务采用的是经典的客户端/服务器服务模型。客户端向服务器发送请求，服务器收到请求并响应请求，返回相关的信息。这种模型使得WWW服务更加灵活、高效、可靠，并能够满足不同用户的不同需求。

简答及计算题

  1. 分析IP地址数量
    已知地址：140.120.84.24/20
    那么根据CIDR地址块，可知前20位不变。由于每个字节占8位，于是140.120不变。
    84=64+16+8，即84=01011000，即前4位0101不变。
    最小地址：140.120.80.0
    最大地址：140.120.95.255
    地址总数：16×256=4096
    相当于16个C类网。

  2. 数据报分片原理
    • 首先，我们需要确定每个数据报片的数据字段长度。由于固定首部长度为20字节，那么每个数据报片的数据部分最多只能是1500-20=1480字节。

• 然后，我们需要确定需要划分为几个数据报片。由于原始数据报的数据部分有4000-20=3980字节，那么我们可以用3980除以1480得到2.689，向上取整得到3，即需要划分为3个数据报片。

• 接下来，我们需要确定每个数据报片的片偏移字段和MF标志位。第一个数据报片相对于原始数据报没有偏移，所以片偏移字段为0，而且后面还有其他的数据报片，所以MF标志位为1。第二个数据报片相对于原始数据报偏移了1480字节，所以片偏移字段为1480/8=185，而且后面还有其他的数据报片，所以MF标志位为1。第三个数据报片相对于原始数据报偏移了1480+1480=2960字节，所以片偏移字段为2960/8=370，而且后面没有其他的数据报片了，所以MF标志位为0。

综上所述，我们可以得到以下答案：

• 路由器应当将此数据段划分为3个数据报片。

• 各数据报分片的数据字段长度分别为1480字节、1480字节和1040字节。

• 各数据报分片的片偏移字段分别为0、185和370。

• 各数据报分片的MF标志位分别为1、1和0。

    （1）ARP查询报文：根据缺省网关的IP地址查询其MAC地址
    （2）DNS查询报文：用来向DNS服务器请求解析 www.qq.com 的IP地址
    （3）HTTP请求报文：这种报文是用来向目标网站请求具体的资源或服务的，比如网页、图片、视频等

  2. CSMA/CD协议工作原理
    载波监听：每个节点在发送数据之前，都要先检测信道是否空闲，即是否有其他节点正在发送数据。如果信道空闲，则可以发送数据；如果信道忙碌，则要等待信道空闲后再发送。
    多路访问：当信道空闲时，每个节点都有平等的机会发送数据，没有优先级的区别。但是，如果有多个节点同时检测到信道空闲，并且同时开始发送数据，就会发生冲突。
    冲突检测：每个节点在发送数据的同时，也要不断监听信道上的信号，以检测是否有冲突发生。如果检测到冲突，就说明有其他节点也在发送数据，导致信号相互干扰
    冲突处理：如果一个节点在发送数据过程中检测到冲突，就要立即停止发送，并向信道发送一个特殊的冲突信号，以通知其他节点发生了冲突。然后，该节点要随机选择一个时间段后退延迟，再次尝试发送数据。如果连续发生多次冲突，该节点就要逐渐增加后退延迟的时间范围，直到成功发送数据或者达到最大尝试次数为止。

  3. 信噪比公式，香农公式
    根据信噪比公式：信噪比=10lg(S/N)和已知条件，知：
    30=10lg(S/N)，可得S/N=1000
    根据香农公式：C=Wlog2(1+S/N)和已知条件，知：（W是带宽）
    C=4K×log2(1+1000)≈40Kb/s
